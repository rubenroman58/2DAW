Ejercicio 1

1 Modelo de eventos tradicional

En este modelo, se utiliza el atributo onClick, onChange, entre otros, directamente en el HTML. Este modelo vincula los eventos con los elementos HTML de manera declarativa. Es simple pero puede generar problemas de mantenimiento cuando se tiene un código más complejo.

2 Modelo de eventos moderno (addEventListener)

Este modelo permite asignar múltiples controladores a un solo evento de un elemento, usando el método addEventListener. Este modelo es más flexible y permite mayor control sobre los eventos, ya que también permite especificar si el evento se debe manejar en fase de captura o burbujeo.

3 Modelo de eventos en fase de captura

En la fase de captura, el evento se propaga desde el elemento raíz (documento) hasta el elemento objetivo (el que disparó el evento). Esta fase ocurre antes que la fase de burbujeo, donde el evento se propaga hacia arriba desde el objetivo. Es útil cuando se necesitan capturar eventos antes de que lleguen al objetivo.
