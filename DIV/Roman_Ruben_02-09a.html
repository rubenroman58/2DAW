Ejercicio 1

Los formularios son elementos de HTML que permiten a los usuarios enviar datos a un servidor. Funcionan mediante la recopilación de información a través de distintos tipos de campos, como texto, selección, casillas de verificación y más. Cuando el usuario completa el formulario y lo envía, los datos se transmiten al servidor para su procesamiento.

Ejercicio 2

GET y POST son métodos utilizados para enviar datos desde un formulario a un servidor. GET envía los datos a través de la URL, lo que lo hace visible y adecuado para solicitudes simples que no alteran el estado del servidor. Por otro lado, POST envía los datos en el cuerpo de la solicitud, lo que permite transferir información más grande y sensible de manera más segura. Las ventajas de GET incluyen su simplicidad y el uso en enlaces, mientras que POST ofrece mayor seguridad y capacidad para manejar datos complejos.

Ejercicio 6

  1. formaction: Este atributo especifica la URL a la que se enviará el formulario cuando se haga clic en el botón. Si no se proporciona, el formulario se enviará a la URL del atributo `action` del elemento `
    `.
  2. formethod: Define el método HTTP que se utilizará al enviar el formulario. Puede ser "get" o "post". Si no se establece, se usará el método definido en el elemento ``.
  3. formenctype: Indica cómo se codificarán los datos del formulario al enviarlos. Este atributo se aplica solo si el método es "post". Los valores comunes son "application/x-www-form-urlencoded", "multipart/form-data" y "text/plain".
  4. formnovalidate: Si se incluye este atributo en un botón de tipo "submit", evita que el formulario se valide antes de ser enviado. Esto permite enviar formularios incluso si hay campos requeridos o con errores de validación.
  5. formtarget: Especifica dónde se abrirá el documento al enviar el formulario. Los valores comunes incluyen "_self" (abre en la misma ventana), "_blank" (abre en una nueva ventana) y "_parent" (abre en el marco padre).
